# MMTLS 加密协议分析

## 概述

MMTLS 是微信自研的移动 TLS 协议，基于 TLS 1.3 裁剪 + 自定义扩展。

## 加密算法栈

| 算法 | 地址 | 用途 |
|------|------|------|
| X25519 | 0x1864befc0 | 密钥交换 |
| Camellia | 0x1851706bc | 对称加密 |
| SHA256 | 0x1833aa160 | 哈希 |
| SHA512 | 0x18512bda0 | 高强度哈希 |
| RC4 | 0x18343b480 | 遗留流加密 |
| Montgomery | 0x184e5e623 | 大数运算 |
| GF(2^m) | 0x186703348 | 伽罗瓦域运算 |

## 握手流程

```
客户端                              服务器
   │                                  │
   ├─→ ClientHello ─────────────────→ │
   │   (X25519 公钥)                  │
   │                                  │
   │ ←─────────────────── ServerHello ┤
   │    (X25519 公钥 + 证书)          │
   │                                  │
   ├─→ Certificate Verify ──────────→ │
   │   (签名验证)                      │
   │                                  │
   │ ←────────────────── Finished ────┤
   │                                  │
   │   ════════════════════════════════
   │         加密通信开始
   │   ════════════════════════════════
```

## 证书

微信嵌入了 4 个 Ed25519 X.509 证书，用于服务器验证。

## 会话复用

MMTLS 支持 PSK（Pre-Shared Key）会话复用：

- 会话票据有效期管理
- 服务端可远程清除所有 PSK（通过 MMTLSCtrlInfo）

## 安全考虑

1. **不自建 MMTLS 连接** — 使用微信原生连接
2. **复用现有会话** — 不创建新会话
3. **监控 TLS 事件** — 通过回调监控
